     105             : /*!
     106             :  * \ingroup DataBoxGroup
     107             :  * \brief Marks a DataBoxTag as being a compute item that executes a function
     108             :  *
     109             :  * \details
     110             :  * Compute items come in two forms: mutating and non-mutating. Mutating
     111             :  * compute items modify a stored value in order to reduce the number of memory
     112             :  * allocations done. For example, if a function would return a `Variables` or
     113             :  * `Tensor<DataVector...>` and is called every time step, then it would be
     114             :  * preferable to use a mutating compute item so that the values in the already
     115             :  * allocated memory can just be changed.
     116             :  * In contrast, non-mutating compute items simply return the new value after a
     117             :  * call (if the value is out-of-date), which is fine for infrequently called
     118             :  * compute items or ones that do not allocate data on the heap.
     119             :  *
     120             :  * A compute item tag contains a member named `function` that is either a
     121             :  * function pointer, or a static constexpr function. The compute item tag
     122             :  * must also have a `label`, same as the DataBox tags, and a type alias
     123             :  * `argument_tags` that is a typelist of the tags that will
     124             :  * be retrieved from the DataBox and whose data will be passed to the function
     125             :  * (pointer). Mutating compute item tags must also contain a type alias named
     126             :  * `return_type` that is the type the function is mutating. The type must be
     127             :  * default constructible.
